Sysinternals Autologon

April 26, 2026 · 4 min read

Autologon 是 Sysinternals 提供的一个小工具，用来配置 Windows 内置的自动登录机制。配置完成后，系统启动时会使用你提供的凭据自动登录指定用户，而不再等待手动输入用户名和密码。

下载​

  • Autologon - Sysinternals | Microsoft Learn
  • 下载大小: 495 KB
  • 也可以直接通过 Sysinternals Live 运行

功能说明​

Windows 会使用通过 Autologon 输入的凭据执行自动登录，这些凭据会以加密形式保存到注册表中。

安全警告​

尽管密码在注册表中作为 LSA 机密加密，但具有管理权限的用户仍然可以轻松检索并解密密码。

如果机器存在多人共用、管理员权限分散或安全要求较高的场景，不建议开启自动登录。

使用方法​

  1. 运行 autologon.exe
  2. 在弹出的对话框中填写用户名、域和密码
  3. 点击 Enable
  4. 下次系统启动时，Windows 会尝试使用这些凭据在控制台上自动登录用户

注意事项​

  • Autologon 不会验证你提交的凭据是否正确
  • Autologon 也不会验证指定用户账户是否被允许登录到这台计算机
  • 若要关闭自动登录，点击 Disable
  • 如果在系统执行自动登录前按住 Shift 键，本次登录会临时跳过自动登录
  • 还可以通过命令行参数传入用户名、域和密码
autologon user domain password
  • 如果系统启用了 Exchange ActiveSync 密码限制，Windows 将不会处理自动登录配置

原文​

作者: Mark Russinovich

发布时间: 2016 年 8 月 29 日

Autologon 使你能够轻松配置 Windows 的内置自动登录机制。Windows 使用通过 Autologon 输入的凭据（已在注册表中加密）自动登录指定的用户，而不用等待用户输入其名称和密码。

尽管密码在注册表中作为 LSA 机密加密，但具有管理权限的用户可以轻松检索和解密密码。

Autologon 使用起来很简单。只需运行 autologon.exe，填写对话框，然后点击“启用”就可以了。下次系统启动时，Windows 会尝试使用输入的凭据在控制台上登录用户。请注意，Autologon 不会验证提交的凭据，也不会验证指定的用户帐户是否允许登录到计算机。

若要关闭自动登录，请点击“禁用”。此外，如果在系统执行自动登录之前按住 Shift 键，则将为该登录禁用自动登录。还可以将用户名、域和密码作为命令行参数传递:

autologon user domain password

注意: Exchange Activesync 密码限制到位后，Windows 不会处理自动登录配置。
